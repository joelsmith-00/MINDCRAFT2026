# 🏗️ LUMI Professional Architecture (5-Folder Structure)

This document reflects the high-level, professional software architecture of LUMI, categorized by layer and purpose.

---

## 📂 1. `app/` — User Interface Layer
Handles everything the user "sees" and "hears" visually and through launchers.

| File | Purpose |
|:--- |:--- |
| `gui.py` | Main Arc Reactor UI and HUD logic. |
| `wave_ui.py` | Animated voice ripple visualizer. |
| `launcher/Activate_Voice_Mode.bat` | One-click start for Voice Mode. |
| `launcher/Start_Lumi_Listener.bat` | Starts the background "Hey Lumi" listener. |

---

## � 2. `core/` — System Brain Logic
The "Central Nervous System" of the assistant.

| File | Purpose |
|:--- |:--- |
| `engine.py` | **The Brain.** Dedicated Groq AI (LLaMA 3.3) processing. |
| `voice.py` | **Sensory Input.** Manages Microphone (Audio-to-Text) and Speakers. |
| `registry.py` | **The Nervous System.** Recursively loads all features and tools. |
| `skill.py` | **The DNA.** Abstract base class for all assistant capabilities. |
| `memory.py` | **Long-term Memory.** Persistent storage of user facts. |
| `lumi_listener.py` | Wake-word detection logic. |

---

## 📂 3. `features/` — Assistant Capabilities
Grouped by category for professional management.

### 🤖 **automation/**
*   `whatsapp.py` — Messaging via selenium.
*   `email.py` — IMAP email checking.
*   `n8n.py` — Cloud workflow triggers.

### 🖥️ **system/**
*   `system_ops.py` — Volume, Apps, and PC control.
*   `file_ops.py` — Advanced filesystem management.
*   `screenshot.py` — Desktop screen captures.
*   `camera.py` — Webcam photo capture.
*   `web_ops.py` — Google searching and browser info.

### 🧠 **ai/**
*   `vision.py` — Standalone Live Vision orchestrator.
*   `detection.py` — YOLOv8 object detection logic.
*   `gemini_live.py` — Multimodal streaming launcher.
*   `text_summary.py` — AI-powered document summarization.
*   `video_system.py` — The standalone detection engine.

### 🛠️ **utilities/**
*   `datetime.py` — Time and date management.
*   `weather.py` — Real-time weather forecasting.

---

## 📂 4. `integrations/` — External Services & Quality Control
Connectors and testing suites.

| Folder / File | Purpose |
|:--- |:--- |
| `n8n/` | Diagnostic scripts and results for n8n webhooks. |
| `llm/` | Model lists and connectivity tests for Gemini/Groq. |
| `diagnostics/` | Verification scripts (`verify_*.py`) and system checks. |
| `tests/` | Unit tests for WhatsApp and GUI components. |

---

## 📂 5. `assets/` — Data & Storage
All static files and persistent data generated by the user.

*   **models/**: Contains `yolov8n.pt` (Detection brain).
*   **storage/**:
    *   `camera_photos/` — All webcam snaps.
    *   `screenshots/` — All screen captures.
    *   `contacts.json` — WhatsApp phonebook.
    *   `memory.json` — Stored AI memories.
    *   `learn.txt` — Narrative learning log.
    *   `inccomplit.txt` — Task list.

---

## 🚀 Root Files
*   `main.py` — The entry orchestrator.
*   `requirements.txt` — Python dependency manifest.
*   `.env` — Protected API key storage.
*   `README.md` — User manual.
